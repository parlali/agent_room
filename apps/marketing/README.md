# Marketing

Public marketing site for Agent Room.

## Development

```bash
bun run marketing:dev
```

The dev server runs on port `3100` and serves:

- React routes for `/`, `/features`, `/pricing`, `/security`, `/source`, `/terms`, and `/privacy`
- Product screenshots from `public/assets/marketing/`
- Generated OG share image from `packages/brand/exports/marketing/` after `brand:export:marketing`
- Waitlist submissions at `POST /api/waitlist`

## Build and preview

```bash
bun run brand:export:marketing
bun run marketing:build
bun run marketing:preview
```

`marketing:preview` serves `dist/` with the waitlist API on port `4173` by default.

## Environment variables

| Variable                        | Default                                | Purpose                                          |
| ------------------------------- | -------------------------------------- | ------------------------------------------------ |
| `VITE_ALLOWED_HOSTS`            | none                                   | Extra dev-server hostnames, comma-separated      |
| `VITE_WAITLIST_API_URL`         | `/api/waitlist`                        | Waitlist POST endpoint for the browser form      |
| `MARKETING_WAITLIST_DB`         | `apps/marketing/.data/waitlist.sqlite` | SQLite database for waitlist submissions         |
| `MARKETING_WAITLIST_RATE_LIMIT` | `8`                                    | Max submissions per client IP per hour           |
| `PORT`                          | `4173`                                 | Port for `marketing:preview` / `marketing:start` |

### OG share image

Generated by the brand package and not committed:

```bash
bun run brand:export:marketing
```

Writes `packages/brand/exports/marketing/og-share.png`. The marketing Vite build emits it to `dist/assets/marketing/og-share.png` through `@agent-room/brand/vite`. Docker runs the export step before build.

## Waitlist storage

Submissions are stored in SQLite, not email. Inspect records with:

```bash
sqlite3 apps/marketing/.data/waitlist.sqlite "select id, created_at, email, company, interest from waitlist_submissions order by id desc limit 20;"
```

The form includes a honeypot field and server-side rate limiting. Bots that fill the honeypot receive a success response without persisting data.

## Docker

The root `Dockerfile` builds the marketing site into `apps/marketing/dist/` alongside the self-hosted app. Marketing static assets and favicons are included in that build output.
